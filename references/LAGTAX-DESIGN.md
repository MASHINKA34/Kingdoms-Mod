# Налог на лаги — дизайн-документ

Модуль для Kingdoms: профайлер заводов + экономическое ограничение нагрузки + платная прогрузка чанков.
Стек: NeoForge 1.21.1, Create 6.0.10, Kingdoms (клеймы, казна, карта, уведомления), Create Numismatics (шпоры).

## Суть

Spark показывает лаги админу в технических терминах. Здесь лаги становятся игровой механикой:
каждая фракция видит нагрузку своих заводов в понятных цифрах (мс/тик) и платит за неё шпорами.
Сервер регулирует нагрузку экономикой, а не запретами админа. Игроки сами оптимизируют заводы,
потому что оптимизация = экономия денег фракции.

## 1. Профайлер (фундамент)

**Измерение.** Один mixin в тик block entities чанка (`LevelChunk$BoundTickingBlockEntity#tick`
или обёртка над `BlockEntityTickInvoker`): замер `System.nanoTime()` вокруг тика, агрегация в
`Long2LongOpenHashMap` по `ChunkPos`.

**Оверхед.** Полный замер не каждый тик, а выборочно — каждый 4-й тик (25% сэмплинг),
результат экстраполируется. Сглаживание — EMA (экспоненциальное скользящее среднее) с окном ~5 минут.
Детализация до конкретной машины (какой BE в чанке самый тяжёлый) — только по запросу админа
на конкретный чанк, временно, чтобы не платить оверхед постоянно.

**Атрибуция.** ChunkPos → клейм → фракция (уже есть в Kingdoms). Неклеймленные чанки — в общую
категорию «дикие» (видит только админ).

**Интерфейс — предметы, не команды.**
Два предмета без крафта, каждый открывает GUI-экран со статистикой (текстуры 16×16 рисуем сами):

- **Админский анализатор** (выдаётся только через /give или креатив) — топ-10 чанков сервера:
  мс/тик, координаты (клик — телепорт), фракция-владелец; вкладка агрегации по фракциям;
  клик по чанку — временная детализация по машинам внутри.
- **Измеритель фракции** (продаётся у торговца королевства) — нагрузка своей фракции:
  всего мс/тик, квота, разбивка по чанкам, прогноз налога, статус прогрузки чанков и таймеры.

## 2. Налог на лаги

**Бесплатная квота.** Каждой фракции — N мс/тик бесплатно **суммарно по всем её чанкам**
(старт: 2.0 мс, конфиг). Квота и налог считаются на фракцию целиком, не на отдельный чанк;
разбивка по чанкам видна в измерителе и в чеке. Новички и скромные заводы не платят никогда.
Возможный бонус: квота растёт с уровнем/размером фракции.

**Тариф.** Сверх квоты — прогрессивная цена, чтобы мягко ограничивать гигантов:

| Превышение квоты | Цена за 0.1 мс/тик в игровой день |
|---|---|
| 0–2 мс | 100 шпор |
| 2–5 мс | 250 шпор |
| 5+ мс | 600 шпор |

(все цифры в конфиге, правятся на живом сервере)

**Честность.** Налог начисляется за время, когда чанки реально тикали: интеграл мс × время.
Завод в непрогруженном чанке, пока никого нет рядом, не тикает — и не платит. Пока включена
платная прогрузка, чанки тикают 24/7 — и деньги за их нагрузку списываются всё это время,
даже если вся фракция оффлайн.

**Списание.** Раз в игровой день из казны фракции (Numismatics-счёт фракции). В чат офицерам —
чек: сколько списано и топ-3 тяжёлых чанка с координатами (сразу видно, что оптимизировать).

**Неплательщики — отключение сразу, без отсрочек и накопления долга.**
Казна не покрывает счёт → немедленно:
1. Заморозка чанков фракции с заводами (BE не тикают вообще, как в выгруженном чанке —
   без заторов и потерь, машины просто стоят).
2. Платная прогрузка отключается, покупка новой блокируется.
3. Красное уведомление всем членам фракции (оффлайн — через очередь).

Неоплаченный счёт сохраняется как «счёт к оплате». Как только в казне хватает — списывается
автоматически, заводы размораживаются, прогрузка снова доступна.

**Hard cap (опция, конфиг).** Абсолютный потолок мс/тик на фракцию независимо от денег
(защита от «богатых китов», которым налог безразличен). По умолчанию выключен.

Важно: никакого тик-троттлинга работающих заводов — только полная заморозка чанков
неплательщиков. Троттлинг ломает ленты/рецепты, заморозка — нет.

## 3. Уведомления

Все — через существующую систему уведомлений Kingdoms; для оффлайн-игроков — очередь,
доставка при следующем входе.

| Событие | Кому | Как |
|---|---|---|
| 75% квоты (суммарно по фракции) | офицеры | жёлтое в чат при входе: «Заводы едят 1.6/2.0 мс — дальше платно» |
| Превышение квоты | офицеры | раз в игровой день: «Налог: X мс сверх квоты → Y шпор/день» |
| Списание налога | офицеры | чек: сумма + топ-3 чанка |
| Нет денег → заморозка заводов и отключение прогрузки | все члены | красное + маркер на карте (Xaero highlight) + оффлайн-очередь |
| Счёт оплачен → разморозка | все члены | зелёное в чат |
| Прогрузка истекает через 1 ч | офицеры онлайн | actionbar/чат |
| Прогрузка истекла | офицеры | оффлайн-очередь + лог фракции |

## 4. Платная прогрузка чанков

UI уже есть (выделение чанков через карту в Kingdoms) — добавляется оплата и таймер.

- **Тариф:** `цена_за_чанк_час` в конфиге. Пресеты в UI: 12 ч (~1000 шпор) и 24 ч (~2000 шпор)
  за чанк, точные цены калибруются.
- **Таймер — реальное время (wall clock).** Хранится `expiresAt` (epoch millis). Тикает и когда
  игрок оффлайн, и когда сервер выключен. Проверка раз в минуту + при старте сервера.
- **Продление:** доплата суммирует время, максимум 7 дней вперёд (конфиг).
- **Автопродление (опция фракции):** галочка «продлевать из казны» — при истечении списывает
  само, если деньги есть.
- **Синергия с налогом:** при покупке прогрузки показывается текущая тяжесть чанка и прогноз
  налога («этот чанк ест 1.2 мс — прогрузка 24 ч ≈ 2000 шпор + налог сверху»). Возможное
  развитие: цена прогрузки = база + коэффициент × мс/тик чанка.
- Неоплаченный счёт по налогу → покупка/автопродление прогрузки блокируются до оплаты.

## 5. План полной реализации (один релиз, всё сразу)

Порядок разработки — снизу вверх, каждый шаг опирается на предыдущий:

1. **Ядро профайлера**: mixin в тик BE, агрегация по чанкам, сэмплинг, EMA, атрибуция
   чанк → фракция. Хранилище агрегатов в данных фракции.
2. **Учёт времени**: интеграл мс × протиканное время по фракциям (основа счёта за период).
3. **Налоговый цикл**: расчёт по квоте и ступеням тарифа, списание раз в игровой день из
   казны, «счёт к оплате» при нехватке, автосписание при пополнении.
4. **Заморозка**: флаг на чанке, BE в замороженных чанках не тикают; включается при неоплате,
   снимается при оплате. Маркер на карте.
5. **Платная прогрузка**: оплата чанк×время, `expiresAt` по wall clock (тикает оффлайн и при
   выключенном сервере), пресеты 12ч/24ч в UI карты, продление, автопродление из казны,
   блокировка при неоплаченном счёте.
6. **Уведомления**: все события из таблицы §3, оффлайн-очередь.
7. **Предметы и GUI**: админский анализатор + измеритель фракции, экраны статистики,
   текстуры 16×16 (рисуются в ходе реализации), позиция у торговца королевства.
8. **Тесты**: юнит на расчёт тарифа/интеграла; интеграционный прогон в dev-мире с Create —
   заморозка/разморозка, рестарты сервера (персист счёта и таймеров), фракция полностью оффлайн.

Релиз один — сразу вся система. Анонс игрокам заранее; первые дни цены можно крутить
hot-reload'ом по живым цифрам.

## 6. Технические заметки

- Реализуется как новый пакет в Kingdoms (`com.geydev.kalfactions.profiling` и `...tax`),
  а не отдельный мод: клеймы, казна, карта, конфиг и уведомления уже здесь — отдельному моду
  понадобился бы API-мост в Kingdoms без выгоды.
- Персист: агрегаты нагрузки и «счёт к оплате» — в данные фракции; `expiresAt` прогрузки —
  в данные клейма. Всё переживает рестарт.
- Предметы: без крафта; админский — только /give или креатив, игрокам не выпадает;
  измеритель фракции — ассортимент торговца королевства. GUI — свои Screen/Menu по образцу
  существующих экранов Kingdoms, данные с сервера пакетами.
- Конфиг (server.toml Kingdoms): квота, ступени тарифа, интервал списания, hard cap,
  цены прогрузки (за чанк-час + пресеты), лимит чанков прогрузки на фракцию, сэмплинг
  профайлера, цена измерителя у торговца.
- Все цены/пороги — hot-reload без рестарта, чтобы калибровать на живом сервере.
